Self-Service Portal

Situation
Clients need needs various departments to administer Dynatrace on their own without giving Monitoring Admin permissions for frequent requests/tasks.

  • Maintenance Window Administration
  • Alerting Profiles
  • Problem Notification Integration
Complication
Accessing the global configuration settings to perform these tasks aren't possible if you want to modify a specific entity that pertains to your app team. The only way to access these settings is by being granted the Monitoring Admin permission which isn't advisable as it grants permission environment wide.
Resolution
Self-Service Portal solves this by utilizing Dynatrace's APIs and provides an easy to use interface for those configuration screens. Currently the Self-Service Portal supports CRUD operations for Maintenance Windows, and plans on introducing configuration capabilities for Alerting Profiles and Problem Notification Integration.

Development Approach

The Self-Service Portal is an easy to use front-end for interacting with Dynatrace's APIs.

Note that this project is still a work-in-progress and not fully released.

Updates to the Self-Service Portal can be found here.

The front end is based off of SB Admin 2 which is a free, open source, Bootstrap 4 admin theme. The back end utilizes the Python3-API-Framework developed by Aaron Philipose.
